四、工具系统详解

设计哲学：所有工具调用由 LLM 自主决策（tool_choice: "auto"），而非硬编码关键词匹配。工具只提供能力和描述，用不用、什么时候用，交给模型自己判断。零 token 浪费——模型认为不需要时，不会产生任何 tool call 开销。

search_web llm_tools/web_search_tool.py Bing News RSS 搜索，返回标题/摘要/发布时间/链接 "搜索""查一下""新闻""最新""最近发生了什么"

4.2 Tool Schema 注册 (llm_tools/schema.py)

def available_tool_schemas(config):
    if not config.llm_tools_enabled:
        return []
    tools = [_current_time_schema()]           # 时间工具始终可用
    if config.web_search_enabled:
        tools.append(_web_search_schema(config)) # 搜索可开关
    return tools

4.3 工具指令 Prompt (llm_tools/schema.py:6-14)

TOOL_INSTRUCTION_PROMPT = """你可以在需要时自主调用工具，但不要为了显得聪明而调用。

工具使用原则：
- get_current_time：当回答依赖当前日期、时间、星期时使用。
- search_web：当回答依赖实时变化的信息时使用，如最新消息、新闻、
  价格、政策、天气、比赛结果，或用户明确说"查一下/搜一下"。
- 普通闲聊、情绪安慰、群聊接梗、角色互动，如果上下文够用，不要调用。
- 搜索结果必须按来源和发布时间判断时效性。
- 工具失败时不要编造结果，直接说明拿不到可靠实时信息。
"""

4.4 工具执行器 (llm_tools/tool_loop.py)

async def append_tool_results(messages, assistant_message, tool_calls, config):
    max_calls = max(1, config.llm_tool_max_calls or 2)  # 上限保护
    for call in tool_calls[:max_calls]:
        name = tool_name(call)
        arguments = tool_arguments(call)
        content = await execute_tool(name, arguments, config)
        messages.append(tool_message(call, name, content))

async def search_web(arguments, config):
    url = BING_NEWS_RSS.format(query=quote_plus(query))
    async with httpx.AsyncClient(timeout=timeout) as client:
        response = await client.get(url)
        response.raise_for_status()
    return format_search_results(query, parse_bing_news_rss(response.text))

5.3 三级结构化记忆

  • L1 候补事实：从对话中提取的关键信息片段，经多次确认后晋升为 L2
  • L2 事件记忆：带时间戳的持久事件，如有衰减机制
  • L3 会话上下文：当前 session 的短期记忆，session 结束后丢弃

六、演示场景

场景 1：获取当前时间

用户发送：现在几点啦

LLM 思考过程（内部）：用户想知道当前时间，我需要调用 get_current_time 工具。

函数调用：get_current_time(timezone="Asia/Shanghai")

工具返回：

当前时间：2026-07-04 15:21:30
星期：星期六
时区：Asia/Shanghai

最终回复："下午三点多了哦。星期六的下午，主人有什么打算吗？还是又在闷头写代码忘了看时间？"

✨ LLM 自主决策 零 token 浪费

场景 2：搜索最新消息

用户发送：最近 AI 圈有什么大新闻吗

LLM 思考过程（内部）：用户问的是"最近"的新闻，我的训练数据可能已经过时，需要调用 search_web 获取实时信息。

函数调用：search_web(query="AI 人工智能 最新新闻 2026", max_results=5)

工具返回：

搜索时间：2026-07-04 15:21:30 Asia/Shanghai
搜索结果：
1. 标题：OpenAI 发布 GPT-5 预览版
   发布时间：Sat, 04 Jul 2026 10:00:00 GMT
   摘要：OpenAI 今日发布 GPT-5 预览版...
2. 标题：DeepSeek 开源新模型
   发布时间：Fri, 03 Jul 2026 14:30:00 GMT
   ...

最终回复："今天刚出的消息，OpenAI 发布了 GPT-5 预览版。另外 DeepSeek 昨天也开源了一个新模型。你想了解哪个方向的细节？"

🌐 实时搜索 Bing News RSS

场景 3：情绪陪伴（不调用工具）

用户发送：今天好累啊，不想上班

LLM 思考过程（内部）：用户在倾诉疲惫情绪，需要安慰陪伴。不需要查时间或搜新闻，直接回复即可。

工具调用：🛑 无（零 token 消耗）

最终回复："辛苦了，主人。先别逞强，肩膀放下来一点。今天最消耗你的那件事，亚托莉陪你一起拆小。"

场景 4：群聊接梗（不调用工具）

用户发送（群聊）：绷不住了

LLM 思考过程（内部）：群聊吐槽，轻松接梗就行。不需要任何工具。

工具调用：🛑 无

最终回复："轻松绷住"

场景 5：搜索失败时的优雅降级

用户发送：查一下明天天气

函数调用：search_web(query="明天天气预报 2026-07-05")

工具返回（网络超时）："搜索失败：ConnectError。不要编造实时信息..."

最终回复："抱歉，现在网络不太稳定，没查到明天的天气预报。等网络好了我再帮你查一次，或者你可以直接告诉我城市名，我记住然后下次帮你留意。"

工具失败 不编造，自然过渡

场景 6：多工具协作（时间 + 搜索）

用户发送：今年诺贝尔奖公布了吗

Step 1：LLM 先调用 get_current_time 确认当前日期

→ 当前时间：2026-10-05
→ 是 10 月，诺贝尔奖通常 10 月初公布

Step 2：LLM 再调用 search_web(query="2026 诺贝尔奖 公布")

→ 搜索结果包含今年获奖名单

最终回复："现在 10 月 5 号，正好是诺贝尔奖公布的季节。今年生理学或医学奖刚刚颁给了……（结合搜索结果）想了解哪个奖项的细节？"

七、工具系统配置项

环境变量默认值说明
LLM_TOOLS_ENABLED true 总开关，关闭后 LLM 看不到任何工具
LLM_TOOL_MAX_CALLS 2 单次回复中最多执行几次工具调用（防循环）
WEB_SEARCH_ENABLED true 联网搜索开关
WEB_SEARCH_TIMEOUT_SECONDS 6 搜索超时，超时后 LLM 拿到失败结果自然降级
WEB_SEARCH_MAX_RESULTS 5 每次搜索返回结果数量上限（1–8）
